# TaskAI 開發協作規範

最後更新：2026-03-28

本文件用來記錄目前專案的協作與交付規範，避免後續開發時出現口頭約定遺漏、文件不同步或資料庫腳本被直接覆寫的情況。

## 1. 回覆語言

- 後續所有與專案相關的回覆、說明與交付摘要，一律使用繁體中文。

## 2. 程式變更必須同步更新文件

- 只要有實際變更程式、設定、流程、環境變數、部署方式或功能行為，就必須同步更新 `docs/` 下的相關文件。
- 文件更新應以「最接近實際變更內容」為原則，例如：
  - 環境變數或部署調整：更新 `docs/vercel-deployment.md`
  - 產品流程或功能範圍變更：更新對應產品或規格文件
  - 技術決策、限制、風險或操作方式變更：新增或更新對應技術文件
- 文件只記錄「最後已採用的變更結果」與目前實際生效的行為。
- 不記錄中途嘗試、已撤回方案、暫時性 workaround 或尚未採用的過程描述，除非這些內容已成為正式限制或風險。

## 3. 資料庫 SQL 變更規範

- `docs/db/taskai_v1_schema.sql` 視為基準 schema，不直接修改。
- 後續若有資料表、欄位、索引、RLS、trigger、function 或資料修正需求：
  - 一律在 `docs/db/` 下面新增新的 SQL 檔案
  - 新檔案只放「新增或需要變更的內容」
  - 不覆寫原始 schema 檔

## 4. 建議的 SQL 檔命名方式

- 建議使用日期加語意化名稱，方便追蹤，例如：
  - `2026-03-28_add_task_summary_index.sql`
  - `2026-03-28_update_task_claim_rls.sql`
  - `2026-03-28_backfill_org_invite_codes.sql`

## 5. 交付原則

- 每次完成變更時，應一併說明：
  - 改了什麼
  - 為什麼改
  - 哪些文件已同步更新
  - 若涉及資料庫，新增了哪一個 SQL 檔
